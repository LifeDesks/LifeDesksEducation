ABOUT
-----

This module provides a self-contained default CCK Content Type that includes all of the Species Profile Model (SPM)
data objects supported by the Encyclopedia of Life (EOL) content partner schema (http://services.eol.org/schema/documentation_0_2.pdf).
In addition, this module makes use of Views and Views Bonus Pack to create XML output of 'published' nodes
of this type. View's field labels are used to match the CCK fields to the EOL schema and a theme template
(*tpl.php) file is used to iterate through the fields and place them in the correct position to generate the
final EOL schema.

USING A PRE-EXISTING CONTENT TYPE AND CUSTOMISING VIEW
------------------------------------------------------

Rather than using the content type provided you can also import the EOL fields into an existing content type. Go to
Administer >> Content Management >> EOL Partnership (admin/content/eol-partnership)

If you prefer not to use the EOL fields you can map your own fields to EOL schema in the eol_partnership view. To do this
a user must have 'administer views' permission. Go to Adminster >> Site building >> Views then enable and edit the eol_partnership view.

Add your own pre-existing fields to the view changing the field label in the view to match one of the predefined EOL schema tags, e.g.:
dwc:ScientificName
dwc:Family
agent role="author"

REQUIREMENTS
------------

* CCK
* Views
* Views Bonus Pack

You must register for an EOL Content Partner account: https://eol.org/content_partner/register?method=get in order to
instruct EOL to harvest the XML file produced by this module.

Enable EOL Partnership Organic Groups (OG) module, to add OG support, requirements are:

* EOL Partnership
* Organic Groups

Enable EOL Partnership Workflow module to add Workflow support, requirements are:

* EOL Partnership
* Workflow

Enable EOL Partnership Internationalization module to add language support, requirements are:

* EOL Partnership
* Views Translations (a sub module of Internationalization)


INSTALLATION
------------

1. Place the "eol_partnership" folder in your modules directory e.g. "sites/all/modules".
2. Enable the module under Administer >> Site building >> Modules (admin/build/modules).
3. Go to Administer >> User Management >> Permissions (admin/user/permissions) and give roles the 'administer eol_partnership' permission
3. Go to Administer >> Content Management >> EOL Partnership and add EOL fields to your site
4. (optional) Go to Administer >> Site Building >> Views and edit the eol_partnership view, customising as suggested.
